# 同事 Quick Start：中午吃点啥

状态：受控 Internal Beta。仅在 Stage 7D 计划和同事 cohort 获批、且收到负责人邀请后使用。

管理端地址：[https://lunchserver-production.up.railway.app/](https://lunchserver-production.up.railway.app/)。

## 先准备

- 使用桌面版 Chrome，以及管理员提供的 Extension 压缩包。
- 向所属小组管理员索取当前小组邀请码。
- 本产品使用轻量身份，不是正式账号；显示名不保证唯一。请不要清除已连接设备的浏览器或 Extension 数据，除非已准备好重新连接。

## 1. 安装 Chrome Extension

1. 将管理员提供的 ZIP 解压到固定目录（不要直接从“下载”目录加载），并确认该目录根部有 `manifest.json`。
2. 打开 `chrome://extensions`，开启 **Developer mode**。
3. 点击 **Load unpacked**，选择刚才的解压目录；可将「中午吃点啥（内部测试）」固定到工具栏。
4. 点击 Extension 图标，再打开设置，完成下面的身份和小组连接。

这是内部 unpacked 包，没有自动更新。如需升级，不要删除旧 Extension；在原加载目录替换文件后点击 **Reload**，这样才能保留本机身份、当前小组和缓存。完整安装/升级说明见[内部 Extension 分发说明](extension-internal-distribution.md)。

## 2. 创建或连接身份

可以在管理端或 Extension 设置页完成。

1. 首次使用：输入显示名，点击「继续」创建轻量身份。
2. 在第二台设备使用同一身份：在仍已连接的旧设备点击「生成身份连接码」；在新设备选择「连接已有身份」并输入该码。
3. 连接码仅可使用一次，10 分钟后失效。生成新码会使旧的未使用连接码失效。

若新设备已连接了另一身份，先选择「断开此设备」再输入连接码。邀请码、身份连接码和任何 Token 都是敏感凭证：只私下交给目标同事，不要发到群聊、工单或截图中。

## 3. 加入或创建小组

### 加入已有小组

1. 在管理端选择「创建/加入小组」，或在 Extension 设置页进入小组区域。
2. 在「加入已有小组」中输入管理员给出的 `LUNCH-…` 邀请码。
3. 成功后该小组会成为当前小组；之后可在管理端或 Extension 设置中切换已加入的小组。

### 创建新小组

当前生产环境默认关闭“公开建组”。即使界面显示「创建新小组」，未获开通时会提示无法创建；请联系产品/运维负责人代建或开通权限。

在获准创建的环境中，填写小组名称（可选副标题）并保存首次显示的邀请码。创建者会成为该小组管理员。邀请码的明文只会在创建或轮换时显示一次；同一码会持续有效，直到管理员轮换它。

## 4. 维护餐厅和推荐理由

在管理端打开「餐厅库」：

1. 点击「新增餐厅和首条推荐」。
2. 填写餐厅名称、一道推荐菜和清楚的推荐理由；这三项是首次新增的必填内容。
3. 尽量补充区域、菜系、步行时间、价格和标签；也可补充适合的天气、星期和心情标签。
4. 保存后，餐厅及推荐只属于当前小组。已有餐厅可继续「添加推荐」；Popup 中的「加个新店」也可快速提交同类信息。

普通成员可以维护自己创建的餐厅和推荐；小组管理员可以维护全部内容，并将餐厅设为正常、暂停或避雷。

## 5. 在 Popup 查看和反馈今日推荐

1. 点击 Chrome 工具栏中的 Extension 图标。
2. 若当天尚未生成推荐，按页面提示生成；已有推荐时会显示最多 2–3 个选项。点击卡片可查看推荐理由和评分明细。
3. 需要重新排序时，点击「换一批」。这会生成新的当前批次，旧批次会保留在记录中，请按需要使用。
4. 记录「今天参与」或「今天不吃」；选定餐厅后点击「就决定是你了」。也可提交「想吃 / 不想吃 / 已吃过 / 避雷」反馈。

若服务暂时不可用，Popup 可能显示上次缓存的推荐；缓存仅供查看，恢复网络后请点击重试获取最新推荐。

## 6. 小组管理员日常管理

在管理端的「设置」页面，管理员可以：

- 修改小组名称、办公室城市/时区和坐标；
- 调整 Extension 的小组默认提醒与二次提醒；成员仍可在自己的 Extension 设置中覆盖本机提醒；
- 调整可解释推荐评分权重（只影响之后新生成的推荐）；
- 查看成员及其餐厅、推荐、反馈贡献，设为管理员/普通成员，或移除、恢复成员；
- 在「团队邀请码」中轮换邀请码。轮换会立即使旧码失效，新码只显示一次；
- 维护所有餐厅的状态与内容。

最后一位有效管理员不能被降级或移除。需要换设备时，请先生成身份连接码；如果所有已连接设备的数据都被清除，只能创建新身份并重新加入小组。

## 遇到问题

请向小组管理员反馈 Chrome 版本、Extension 版本、问题发生在哪一步和不含敏感码的截图。不要发送邀请码、身份连接码或 Token。
