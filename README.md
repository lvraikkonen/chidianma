# 中午吃点啥

Chrome MV3 extension + Fastify + PostgreSQL lunch recommendation tool.

## Local quick start

1. Install dependencies:

   ```bash
   pnpm install
   ```

2. Configure server:

   ```bash
   cp apps/server/.env.example apps/server/.env
   ```

3. Run Prisma:

   ```bash
   pnpm --filter @lunch/server prisma:generate
   pnpm --filter @lunch/server prisma:migrate -- --name init
   pnpm --filter @lunch/server prisma:seed
   ```

4. Start server:

   ```bash
   pnpm dev:server
   ```

5. Build extension:

   ```bash
   pnpm --filter @lunch/extension build
   ```

6. Load `apps/extension/dist` in `chrome://extensions`.

7. Start admin locally:

   ```bash
   export VITE_API_BASE_URL=http://localhost:3000
   pnpm dev:admin
   ```

## Railway deploy checklist

1. Create a Railway project with one PostgreSQL service and one app service.
2. Set the variables listed in `apps/server/README.md`.
3. Deploy `apps/server` with:

   ```bash
   pnpm --filter @lunch/shared build
   pnpm --filter @lunch/server prisma:generate
   pnpm --filter @lunch/server build
   pnpm --filter @lunch/server exec prisma migrate deploy
   pnpm --filter @lunch/server start
   ```

4. Copy the Railway public domain into the extension settings as the API base URL.

## Admin local usage

The Admin app runs locally and points at the backend you want to manage. Production Admin static hosting remains deferred to Stage 6; do not create a separate Railway service for `apps/admin` yet.

### Use Admin with the local server

1. Generate/build shared contracts, then start the local server:

   ```bash
   pnpm --filter @lunch/shared build
   pnpm dev:server
   ```

2. In another terminal, start Admin against that server:

   ```bash
   VITE_API_BASE_URL=http://localhost:3000 pnpm dev:admin
   ```

3. Open the Vite URL, usually `http://localhost:5173`.
4. Enter a display name to create a lightweight local identity.
5. Create a group, or join an existing group with its real one-time invite code.
6. Select the active group from the group switcher.
7. Use **今日推荐** to generate/review the current batch and **餐厅库** to maintain restaurants and recommendations.

The Admin never asks for or renders raw identity/group session tokens, and it does not embed `TEAM_INVITE_CODE` in the frontend bundle.

### Use local Admin with the Railway dev API

1. Confirm the Railway server is healthy:

   ```bash
   curl https://your-server.up.railway.app/api/health
   ```

2. Start the local admin app against the Railway server. Do not include a trailing slash in the API base URL.

   ```bash
   VITE_API_BASE_URL=https://your-server.up.railway.app pnpm dev:admin
   ```

3. Open the Vite URL, usually `http://localhost:5173`.
4. Enter a display name, then create a Railway dev group or join one using its real one-time invite code.
5. Select the active group and verify Today and Restaurants against that group.

This is an API smoke only. It does not mean Admin static hosting is complete; that remains Stage 6 work.

### Admin notes

- Group invite codes are generated by the group-creation flow and entered only when joining that group.
- `TEAM_INVITE_CODE` must not be embedded in Admin frontend bundles.
- `EXTENSION_READ_TOKEN` is only for extension/API read access; it is not used to log in to admin.
- The Chrome extension API base URL should also omit the trailing slash, for example `https://your-server.up.railway.app`.
